PRISM2 - API REFERENCE
A GUI abstraction library for building UI kits
================================================================

----------------------------------------------------------------
CRATE: prism2  (lib root — src/lib.rs)
----------------------------------------------------------------

PURPOSE
  Core crate that ties everything together. Defines the runtime
  context, request system, asset loading, and the Listener and
  Clip utility components.

KEY TYPES

  State
    A type-map (backed by anyanymap) used to store arbitrary
    typed values. Components read/write shared state through it.

  Context
    Holds a State and an mpsc Sender<Request>. Passed mutably
    through every event handler so components can send requests
    or read shared state.
      .send(request)               — enqueue any Request
      .register_listener::<T>()    — subscribe to State changes
                                     for type T; returns a Receiver<T>

  Instance
    The runtime owner of the Receiver<Request> end of the
    channel. Drives the event loop:
      .tick(ctx)                   — run all registered listeners
      .handle_requests()           — drain the channel; returns
                                     Hardware variant if one arrives
      .events                      — VecDeque of pending Events

  Request  (enum)
    Messages components send to the runtime:
      Event(Box<dyn Event>)        — inject a synthetic event
      Hardware(Hardware)           — trigger a platform call
      Service(String, String)      — named service call (reserved)
      Listener(Box<dyn Fn(&mut State)>) — register a state watcher

  Hardware  (enum)
    Platform capabilities:
      GetCamera / StopCamera / PhotoPicker
      GetSafeArea
      SetClipboard(String) / GetClipboard
      SetCloud(String,String) / GetCloud(String)
      Share(String)
      Haptic

  StoredHash<T>
    Internal helper stored in State to detect when a watched
    value T actually changes (via hashing), avoiding redundant
    listener callbacks.

  Listener<D, T>  (Component)
    Wraps a drawable D. On every TickEvent it checks a
    Receiver<T> and, if a new value arrived, calls a user-
    supplied closure  Fn(&mut Context, &mut D, T).
    Construction: Listener::new(ctx, inner, updated_on)

  Clip<D>  (Drawable)
    Render-only wrapper. Passes request_size and build through
    to D unchanged. In draw() it intersects the incoming bound
    rect with its own absolute rect, discarding pixels outside.
    No layout impact — pure clipping.

  Assets  (struct, static helpers)
    .load_file(dir, file)   -> Option<Vec<u8>>
    .load_svg(bytes)        -> RgbaImage
    .load_png(dir, file)    -> Option<RgbaImage>

  IS_MOBILE: bool   — true on iOS / Android
  IS_WEB:    bool   — true on wasm32

DEPENDENCIES (notable)
  wgpu_canvas   — low-level GPU canvas (shapes, text, images)
  prism_proc    — derive macros
  anyanymap     — heterogeneous type map for State
  downcast-rs   — downcasting trait objects
  dyn-clone     — Clone across trait objects
  image         — pixel image loading
  nsvg          — SVG rasterisation
  include_dir   — embed asset directories at compile time
  uuid          — unique IDs for selectables / text inputs

FILE: src/drawable.rs
----------------------------------------------------------------

PURPOSE
  Defines the two fundamental traits of the rendering pipeline:
  Drawable (leaf or composite renderable) and Component
  (anything that has children and a Layout).

KEY TYPES

  RequestTree
    Tree of SizeRequests mirroring the component hierarchy.
    Built bottom-up before layout.
      .0  SizeRequest   — this node's size request
      .1  Vec<RequestTree> — children's request trees

  SizedTree
    Tree of resolved sizes and child offsets produced by build().
      .0  Size = (f32,f32)
      .1  Vec<(Offset, SizedTree)>

  type Offset = (f32, f32)
  type Size   = (f32, f32)
  type Rect   = (f32, f32, f32, f32)   — (x0,y0,x1,y1)

  Drawable  (trait)
    Core rendering trait. Every visible element implements this.

    Required:
      .request_size(&self) -> RequestTree
          Report min/max size constraints (bottom-up pass).

      .draw(&self, sized, offset, bound)
               -> Vec<(CanvasArea, CanvasItem)>
          Produce GPU draw calls given the resolved layout.

    Provided (can override):
      .build(&self, size, request) -> SizedTree
          Resolve the actual size from the allotted size and the
          request. Default just calls request.0.get(size).

      .name(&self) -> String
          Debug name (defaults to type name).

      .event(&mut self, ctx, sized, event)
          Handle an incoming event. Default is a no-op.

    Blanket impls:
      Box<dyn Drawable>   — delegates all methods to the inner box
      Option<D>           — renders/handles nothing when None
      Text, Shape, Image  — wgpu_canvas primitives; fixed size,
                            emit a single CanvasItem each

  Component  (trait)
    Implemented by composite components (via #[derive(Component)]).

    Required:
      .children(&self)     -> Vec<&dyn Drawable>
      .children_mut(&mut)  -> Vec<&mut dyn Drawable>
      .layout(&self)       -> &dyn Layout

    Optional:
      .clipped(&self) -> bool          — enable clip rect (default false)
      .clip_origin(&self) -> Option<Offset>

    Blanket impl:
      impl<C: Component + OnEvent> Drawable for C
        Provides the full request_size / build / draw / event
        pipeline for any Component automatically:
          request_size — recurses into children, delegates to layout
          build        — delegates sizes to layout, recurses
          draw         — clips if clipped(), recurses with
                         tightened bounds, skips invisible children
          event        — calls OnEvent::on_event, then routes
                         resulting events to children via
                         Event::pass()

  drawables![ expr, ... ]  (macro)
    Shorthand to create Vec<Box<dyn Drawable>> from a list of
    concrete drawables.

FILE: src/event.rs
----------------------------------------------------------------

PURPOSE
  Defines the event system: the Event trait, all built-in event
  types, the OnEvent handler trait, and supporting enums/structs.

TRAITS

  Event  (trait, object-safe)
    Any value that can propagate through the component tree.
      .pass(self: Box<Self>, ctx, children: &[Area])
               -> Vec<Option<Box<dyn Event>>>
          Returns one slot per child. Some(e) passes the event
          (possibly transformed) to that child; None stops it.
    Implemented with impl_downcast! so handlers can downcast.

  OnEvent  (trait)
    Implemented by components that intercept events.
      .on_event(&mut self, ctx, sized, event)
               -> Vec<Box<dyn Event>>
          Transform / consume / emit events. The returned vec is
          what gets routed to children via Event::pass().
          Default: return vec![event] (pass through unchanged).

BUILT-IN EVENT TYPES

  MouseEvent
    Fields: position: Option<(f32,f32)>, state: MouseState
    position is Some only for the component under the cursor.
    pass() propagates to all children but sets position=None for
    children not containing the hit point (first-hit stops).

  MouseState  (enum)
    Pressed | Moved | Released | Scroll(f32, f32)

  KeyboardEvent
    Fields: key: Key, state: KeyboardState, modifiers: Modifiers
    pass() broadcasts to all children unchanged.

    MODIFIER KEY BEHAVIOUR
    Modifier keys (Control, Shift, Alt, Meta, CapsLock, NumLock,
    ScrollLock) generate KeyboardEvents normally and are tracked
    in held_keys by consumers such as Canvas. However, consumers
    that implement modifier-aware input (e.g. Canvas::on_key_press)
    filter modifier-only keypresses out of their callback and
    GameEvent systems — modifiers only matter as part of a combo.
    Always read modifier state via the modifiers field rather than
    checking for Named(Control) etc. in key match arms.

  KeyboardState  (enum)   Pressed | Repeated | Released

  Modifiers  (struct)
    Fields: shift: bool, ctrl: bool, alt: bool, meta: bool
    Constructors: none() shift() ctrl() alt() meta()
    .is_none() -> bool   true when all fields are false

    Use is_none() to check that no modifier is held.
    Compare two Modifiers with == for exact match.
    Example — reading modifiers in an OnEvent handler:
      if let Some(KeyboardEvent { key, modifiers, .. })
             = event.downcast_ref::<KeyboardEvent>() {
          if modifiers.ctrl {
              // handle Ctrl+key
          }
      }

  Key  (enum)
    Named(NamedKey) | Character(String)

  NamedKey  (enum, non_exhaustive)
    Full set of named keys: Enter, Tab, Space, Backspace,
    Escape, Arrow*, Home, End, PageUp/Down, Delete, Insert,
    Shift, Control, Alt, Meta, CapsLock, NumLock, ScrollLock,
    F1–F12, media keys, etc.

    Modifier NamedKeys (Shift, Control, Alt, Meta, CapsLock,
    NumLock, ScrollLock) are valid Key values but are treated
    as pure state carriers by higher-level input systems.
    Do not match on them in key-press handlers; use the
    modifiers field instead.

  TickEvent
    Emitted every frame. Broadcasts to all children.
    Used for animations, momentum scrolling, listener polling.

  Button  (enum, semantic — from Button emitter)
    Pressed(bool)   — true=pressed, false=released
    Hover(bool)     — true=entered, false=left
    Disable(bool)
    Broadcasts to all children.

  Selectable  (enum, semantic — from Selectable emitter)
    Pressed(id: String, group_id: String)
    Selected(bool)
    Broadcasts to all children.

  Slider  (enum, semantic — from Slider emitter)
    Start(f32)   — drag began at x
    Moved(f32)   — dragged to x
    Broadcasts to all children.

  TextInput  (enum, semantic — from TextInput emitter)
    Hover(bool)
    Focused(bool)
    Edited(Key, Modifiers)
      Key press while focused. Includes the active Modifiers so
      consumers can handle Ctrl+A, Ctrl+C, Ctrl+V, Ctrl+Z etc.
      without needing a separate held-key query.
    Broadcasts to all children.

  NumericalInput  (enum, semantic — from NumericalInput emitter)
    Delete
    Digit(char)
    Char(char)   — '.', '/', ':'
    Broadcasts to all children.

  HardwareEvent  (enum)
    Clipboard(String)
    Camera(Arc<RgbaImage>)
    SafeArea(f32,f32,f32,f32)
    Broadcasts to all children.

  type Events = VecDeque<Box<dyn Event>>

  events![ expr, ... ]  (macro)
    Shorthand to create Vec<Box<dyn Event>>.

FILE: src/emitters.rs
----------------------------------------------------------------

PURPOSE
  "Emitter" wrappers: thin components that translate raw input
  events into higher-level semantic events and pass them down.

COMPONENTS

  Button<D>
    Wraps D. Converts MouseEvent → event::Button.
    Tracks pressed state (field 2) across press/release.
    On mobile, hover events are suppressed.
    Emits:
      Button::Pressed(true)   on press inside bounds
      Button::Pressed(false)  on release (if was pressed)
      Button::Hover(true/false) on mouse enter/leave

  NumericalInput<D>
    Wraps D. Filters KeyboardEvent on Pressed/Repeated.
    Emits:
      NumericalInput::Delete  for Backspace/Delete
      NumericalInput::Digit(c) for '0'–'9'
      NumericalInput::Char(c)  for '.', '/', ':'
    Other keys are dropped.

  Selectable<D>
    Wraps D. Each instance has a UUID (self id) and a group UUID.
    On press inside bounds: sends a global
      Selectable::Pressed(self_id, group_id)
    via ctx.send so all group members hear it.
    On receiving Selectable::Pressed for its group, re-emits:
      Selectable::Selected(true/false)

  Slider<D>
    Wraps D. Tracks dragging state.
    Emits:
      Slider::Start(x)  on first press inside bounds
      Slider::Moved(x)  on move while held
    Releases clear tracking; events outside drag are dropped.

  TextInput<D>
    Wraps Selectable<Self> (uses a fixed UUID group so all text
    inputs share one focus group).
    Optional requires_focus mode gates keyboard events behind
    focus state.
    Emits:
      TextInput::Focused(bool)      on click in/out or selection
      TextInput::Hover(bool)        on mouse enter/leave (unfocused)
      TextInput::Edited(Key, Modifiers)
        On any KeyboardEvent with state Pressed or Repeated while
        focused. Always emits — including modifier combos such as
        Ctrl+A, Ctrl+C, Ctrl+V, Ctrl+X, Ctrl+Z, Ctrl+Shift+Z —
        so consumers receive both the key and the active modifier
        state in one event and never need a separate held-key query.

  Scrollable<D>
    Wraps Momentum<D>. Suppresses taps that are actually scroll
    gestures (y delta ≥ 5 px). On release of a genuine tap,
    re-emits a synthetic Pressed+Released pair so children see
    a click.
    Exposes Deref/DerefMut to Momentum<D>.

  Momentum<D>  (mobile only — IS_MOBILE)
    Wraps D. Tracks touch velocity and applies momentum
    scrolling after finger lift via TickEvent.
    Decay factor: speed *= 0.92 per tick.
    Emits synthetic MouseEvent::Scroll while coasting.
    On desktop the component is transparent (passes events through).

FILE: src/layout.rs
----------------------------------------------------------------

PURPOSE
  Defines the Layout trait and all built-in layout algorithms,
  plus SizeRequest and helper types used by the layout pipeline.

TRAITS

  Layout  (trait)
    .request_size(children: Vec<SizeRequest>) -> SizeRequest
    .build(size: (f32,f32), children: Vec<SizeRequest>) -> Vec<Area>

KEY TYPES

  SizeRequest
    min_width, min_height, max_width, max_height: f32
    Constructors:
      ::new(min_w, min_h, max_w, max_h)
      ::fixed((w,h))   — min==max
      ::fill()         — 0..f32::MAX
    Methods:
      .get(size) -> (f32,f32)   — clamp size into bounds
      .add(w,h) / .add_width / .add_height / .remove_height
      .max(other) / .combine(other)

  Area
    offset: (f32,f32), size: (f32,f32)
    Result of Layout::build for each child.

  Padding(top, left, bottom, right: f32)
    .new(p)              — uniform padding
    .adjust_size(size)   — subtract padding from size
    .adjust_offset(off)  — add top/left to offset
    .adjust_request(req) — add total padding to request

  Offset  (enum)
    Start | Center | End | Static(f32)
    .get(max_size, item_size) -> f32
    .size() -> Option<f32>

  Size  (enum)
    Fit    — shrink-wrap children
    Fill   — expand to available space
    Static(f32)
    Custom(Box<dyn CustomFunc>)
    .get(items, fit_fn) -> (f32,f32)
    ::max(items) / ::add(items)  — fold helpers

  ScrollAnchor  (enum)
    Start (default) | End
    Controls whether Column scroll is anchored to the top or bottom.

BUILT-IN LAYOUTS

  DefaultStack
    Overlays all children at (0,0). Size = max of all children.

  Stack(x_offset, y_offset, x_size, y_size, padding)
    Overlays children, each positioned by x/y Offset within the
    stack's bounds. Constructors: center() start() end() fill()

  Row(spacing, offset, size, padding)
    Horizontal layout. Distributes width via UniformExpand,
    positions children vertically via Offset.
    Constructors: center(sp) start(sp) end(sp)
                  new(spacing, offset, size, padding)

  Column(spacing, offset, size, padding, scroll?, anchor)
    Vertical layout with optional scrolling.
    .adjust_scroll(delta)  — move scroll position
    .set_scroll(val)
    Scroll clamped to (0, content_height - view_height).
    ScrollAnchor::End reverses child order (chat-style).
    Constructors: center(sp) start(sp) end(sp)
                  new(spacing, offset, size, padding, scroll)

  Wrap(w_spacing, h_spacing, x_offset, y_offset, padding)
    Horizontal flow that wraps to new rows when width is
    exceeded. Row x-alignment controlled by x_offset.
    Stores available width in an Arc<Mutex<f32>> so
    request_size can re-run with the actual width after build.
    Constructors: new() start() end() center()

  UniformExpand  (helper)
    ::get(sizes: Vec<(min,max)>, max_size, spacing) -> Vec<f32>
    Greedy algorithm that distributes free space uniformly among
    expandable items (those where min < max), iterating until
    no free space remains or no item can grow.

CRATE: prism_proc  (proc-macro crate — src/lib.rs)
----------------------------------------------------------------

PURPOSE
  Provides the #[derive(Component)] macro used on every
  component struct or enum in Prism.

MACRO: #[derive(Component)]

  Works on structs and enums.

  STRUCT rules:
    - Field 0   must be the Layout (Stack, Row, Column, etc.).
                It becomes the impl of Component::layout().
    - Fields 1+ are children. Each becomes an entry in
                Component::children() and children_mut().
    - #[skip]   on a field excludes it from children (use for
                internal state like booleans, UUIDs, etc.).

  Supported child field types (auto-detected):
    D                    — single Drawable child
    Option<D>            — optional child
    Option<Box<D>>       — optional boxed child
    Box<D>               — boxed child
    Vec<D>               — list of children
    Vec<Box<D>>          — list of boxed children
    HashMap/IndexMap<_,D>— map of children (iterated by value)
    (D0, D1, ...)        — tuple of children (all elements)

  ENUM rules:
    Each variant must have named fields.
    First named field of each variant is its layout.
    Remaining non-#[skip] fields are children.
    Generates match arms for children(), children_mut(),
    and layout().

  Generated impl:
    impl Component for T {
        fn children(&self)     -> Vec<&dyn Drawable>
        fn children_mut(&mut self) -> Vec<&mut dyn Drawable>
        fn layout(&self)       -> &dyn Layout
    }

  Combined with the blanket
    impl<C: Component + OnEvent> Drawable for C
  in drawable.rs, deriving Component gives a type the full
  Drawable implementation for free.


================================================================
QUICK REFERENCE — IMPLEMENTING A NEW COMPONENT
================================================================

1. Define your struct:

   #[derive(Debug, Clone, Component)]
   pub struct MyWidget(
       Column,           // field 0: layout
       Label,            // field 1: child drawable
       Button<Icon>,     // field 2: child drawable
       #[skip] bool,     // internal state, not a child
   );

2. Implement OnEvent to handle events:

   impl OnEvent for MyWidget {
       fn on_event(&mut self, ctx, sized, event) -> Vec<Box<dyn Event>> {
           if let Some(event::Button::Pressed(true)) = event.downcast_ref() {
               // react to button press
           }
           // Handle keyboard with modifier awareness:
           if let Some(KeyboardEvent { key, modifiers, .. })
                  = event.downcast_ref::<KeyboardEvent>() {
               if modifiers.ctrl {
                   // handle Ctrl+key combo
                   return vec![];
               }
               // handle bare key
           }
           // Handle TextInput::Edited which carries modifiers inline:
           if let Some(TextInput::Edited(key, modifiers))
                  = event.downcast_ref::<TextInput>() {
               if modifiers.ctrl {
                   // handle Ctrl+key in text input context
               }
           }
           vec![event]   // pass through
       }
   }

3. That's it. The derive macro + blanket impl give you:
   request_size, build, draw, and event routing automatically.
